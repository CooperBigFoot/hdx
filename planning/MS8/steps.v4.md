## 0. Grounding — the enforced-vs-skipped reconciliation (the CRITICAL fold, done FIRST)

This milestone is grounded in the **actual** `crates/core/src/validate.rs` + the
discovery code (`scalar_reader.rs`, `discovery.rs`, `gridded_discovery.rs`,
`geoparquet_reader.rs`, `manifest.rs`), read in full before planning. The §14 checks
fall into **three buckets**. A "one-violation-per-check INVALID fixture" is built
(programmatically, one surgical mutation off the valid baseline) **only for the checks
that can actually yield `conformant:false`** (Bucket-B). For the rest we do **not**
claim a `conformant:false` negative; we document why none exists in v0.1.

### Bucket-A — Entry-gate `Err` (NOT a `conformant:false` report)

`validate` (validate.rs:1260) reads `manifest.json`, then `Manifest::from_json`
(validate.rs:1275, the early `?`) — whose first acts are the §0/M2 hard cut and the
M3/M4 boundary checks (manifest.rs:17-20) — **before `discover` is ever called**. A
manifest violation therefore returns `Err(ValidateError::Manifest(..))`, never a
report. `build_report` lists M1–M4 as `ran:pass` unconditionally (validate.rs:1373-1375)
**only because the entry gate already cleared them** — that arm is never reached on a
Bucket-A tree (the `?` returns first). It is the already-cleared-at-the-entry-gate
convention, **not** a second enforcement site (noted in S1/S2 + the README).

| Check | Mutation → outcome | Status |
|---|---|---|
| **M2** | `format_version: "0.2"` → `Err(Manifest(UnknownFormatVersion))` | **exists** (MS2 `invalid/wrong-format-version/`) |
| **M3** | a 7th manifest field → `Err(Manifest(ExtraManifestField))` | **new (S2)** |
| **M4** | empty `crs` / empty `cadence` / bad `created_at` → `Err(Manifest(EmptyCrs / EmptyCadence / InvalidTimestamp))` | **new (S2)** |

> **M6 unreachable-by-construction note.** `check_m6` rule (a) (validate.rs:1060-1066)
> would `ran:fail` on an **empty** cadence, but an empty cadence is rejected by **M4** at
> the entry gate (manifest.rs:20: `crs`/`cadence` non-empty), so `validate(empty-cadence)`
> returns `Err(Manifest(EmptyCadence))` **before** `build_report`. Rule (a)'s fail leg is
> therefore **dead code on the validate path**: the empty-cadence fixture pins **M4**, not
> M6. Stated in S2 + the README so no one later adds a spurious M6 empty-cadence negative.

## Ordering rationale

The build is **fixtures → tests → docs**, each step independently committable and green:

1. **S1 (baseline reconciliation, no new fixtures).** Re-affirm the byte-unchanged MS2
   valid baseline and its two existing goldens (`describe.golden.json`,
   `validate.golden.json`) and the four committed snapshot tests still hold, and add the
   one-line "M1–M4 ran:pass is the entry-gate convention, not a second enforcement site"
   note. This is the green floor every later step stands on — MS8 adds **no** field, so the
   baseline goldens are byte-stable. Tiny, reviewable, and proves the regenerate is still
   byte-deterministic before any mutation is added.
2. **S2 — Bucket-A generator + Err-negative tests (M3, M4; M2 already exists).** The
   entry-gate negatives need only manifest mutations (cheapest, no data rewrite); they
   commit a self-contained generator + regression slice.
3. **S3 — Bucket-B scalar/identity/homogeneity/time/layout negatives (I1, I2, I3, H1, T1,
   L2).** The negatives reachable by parquet / layout mutations. Each is one surgical
   mutation + one pinned "exactly this id fails, others pass/skip" regression test + a
   committed `validate.golden.json` per fixture.
4. **S4 — Bucket-B gridded/CRS negatives + the Geo1 / G1 / G3 findings (M5, G2, H2; Geo1
   probe).** The negatives reachable by COG / Zarr mutations, plus the conditional Geo1
   probe and the G1/G3 no-on-disk-negative findings. Sequenced after S3 because the
   gridded writer is the heavier mutation surface.
5. **S5 — Bucket-C still-conformant fixtures + describe goldens (M6 irregular-axis; the
   describe-golden extension).** The MED-1 M6 fixture (skipped, still conformant) and the
   committed **describe** goldens for the irregular-time-axis fixture + one Bucket-B scalar
   invalid (the goldens fold names extending BOTH describe and validate goldens).
6. **S6 — README + golden-update workflow + companion-mask re-assertion.** The check-id →
   fixture/finding matrix, the enforced-vs-skipped split, the regenerate + golden-update
   workflow, and the re-assertion that `{field}_was_filled` / `{source}_{variable}` are
   ordinary in **both** describe and validate. Docs land last so they describe the final
   committed suite.

Dependencies flow S1→S2→S3→S4→S5→S6. Each step leaves `cargo build` + `cargo test` +
`cargo clippy --all-targets -- -D warnings` green and is one conventional commit with the
mandatory patch bump + tag.
